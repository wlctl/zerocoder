# Использование LogAnalyzer (la)

## Загрузка логов

1. Откройте `http://localhost:8888` → страница «Загрузки».
2. Перетащите файлы в dropzone или нажмите для выбора. Можно **несколько файлов
   или zip-архив одновременно**. Выбранные файлы показаны списком (с размером),
   любой можно убрать до загрузки.
3. Нажмите «Загрузить N файл(ов)».
4. Под формой появится per-file результат:
   - `parsed` — успешно (число файлов в архиве / формат);
   - `duplicate` — файл уже загружался ранее (указан существующий `upload_id`);
   - `failed` — ошибка обработки (текст ошибки).
   Дубликат/ошибка одного файла не прерывает остальные.
5. Таблица загрузок и агрегаты обновляются автоматически.

**Лимиты** (из `la.conf`, перекрываются env): `MAX_FILE_SIZE` (входящий файл,
по умолчанию 10GB), `MAX_FILE_COUNT` (число входящих файлов за запрос, zip = 1).
Дедуп — по MD5 входящего файла.

## Таблица загрузок

- Сортировка (filename / размер / загружен / статус) кликом по заголовку.
- Фильтр по имени файла.
- Колонка «сводка / файлы»: для одиночного лога — сводка парсинга
  (`level_counts` и т.п.); для архива — число файлов.
- «Удалить» — удаляет загрузку и все результаты её обработки (каскад).
- Под таблицей — агрегаты: размер хранилища, число загрузок/файлов/записей.

## Просмотр (drill-in)

Клик по строке → просмотр загрузки. В шапке — детали архива (zip: число файлов,
общий размер, first/last ts) или детали одиночного файла.

- **Селектор файлов** с чекбоксами; по умолчанию выбраны все.
- **Поиск**: фильтр по вхождению текста по всем полям (+ опц. исходное `raw`-поле);
  поиск по всем файлам селектора; фильтров может быть несколько, каждый удаляется
  независимо.
- **Таймлайн** (если применимо): min/max дата с перемещаемыми границами.
- **Подсветка строк**: ввод текста (+ опц. кластерные лексемы) + виджет цвета;
  все подходящие строки подсвечиваются.
- **Пофайловые таблицы** по 10 строк (динамически), одна за другой; над таблицей —
  имя файла и краткая сводка мелким шрифтом; таблицу можно закрыть (чекбокс
  снимается в селекторе) и вернуть отметкой.
- **График событий** по каждой таблице с переключаемой группировкой
  месяц/день/час/минута (видеть всплески).
- **Новое окно** (если возможно): файл открывается в новом окне с управлением от
  основной страницы; открытое в окне не показывается на основной (graceful
  fallback, если блокировщики).

## Персистентность

Фильтры и подсветка сохраняются между рестартами процесса (в БД, per upload).
Кнопка «Очистить» сбрасывает фильтры/подсветку.

## Плагины (парсеры и постобработчики)

Парсеры и постобработчики — подключаемые модули (Go `.so`) в `parsers/` и
`postprocessors/` (пути — `LA_PARSERS_DIR` / `LA_POSTPROCESSORS_DIR`).

- Встроенный `text`-парсер и базовый постобработчик всегда есть в хосте (fallback).
- При старте хост сканирует каталоги и загружает все `.so`; плагин с тем же именем
  формата **заменяет** built-in (override по имени).
- **Добавление/модификация**: соберите/положите только модуль `.so` в нужный
  каталог и перезапустите процесс (`sh/stop-la.sh && sh/start-la.sh`).
  - **Не требуется** пересборка основного бинарника.
  - **Не требуется** повторное развёртывание всего продукта (новый ZIP не нужен).
- Если плагина для формата нет — применяется built-in (text / базовый).

Сборка плагина из исходников (при доработке):

```sh
# из каталога backend-исходников:
go build -buildmode=plugin -tags plugin -o parsers/oracle.so   ./parsers/oracle
go build -buildmode=plugin -tags plugin -o postprocessors/odl.so ./postprocessors/odl
# или все сразу: make plugins
```

Ограничения Go-плагинов: только Linux; один тулчейн и одинаковые версии общих
зависимостей (парсеры в том же модуле); выгрузить `.so` в работающем процессе
нельзя — смена через рестарт.

## Поддерживаемые форматы

| Формат     | Источник                         | Парсер-плагин | Постобработчик-плагин |
|------------|----------------------------------|---------------|------------------------|
| oracle     | Oracle alert.log                 | ✓             | ✓ (сессии старт-стоп)  |
| weblogic   | WebLogic .log (`####<...>`)       | ✓             | ✓ (сессии старт-стоп)  |
| wls_stdout | WLS .out / nodemanager            | ✓             | ✓ (сессии старт-стоп)  |
| java       | log4j-style (multiline stack)     | ✓             | base (fallback)        |
| access     | apache common log                 | ✓             | base (fallback)        |
| odl        | Oracle Diagnostic Log             | ✓             | ✓                      |
| text       | fallback (посточный + дата/уровень)| built-in      | base (built-in)        |

## REST API (кратко)

- `POST /api/uploads` (multipart, multi-file) → `{results:[…]}`
- `GET /api/uploads`, `GET /api/uploads/{id}`, `DELETE /api/uploads/{id}`
- `GET /api/files?upload_id=`, `GET /api/files/{id}`, `GET /api/files/{id}/entries`
- `POST /api/files/{id}/postprocess`, `DELETE /api/files/{id}`
- `GET /api/stats`, `GET /api/parsers`
- Viewer: `search`, `timeline`, `lexemes`, `histogram?bucket=…`,
  `filters`/`highlights` (GET/POST/DELETE), `DELETE /api/uploads/{id}/view-state`
- `GET /healthz`
