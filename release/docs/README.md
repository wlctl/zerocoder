# LogAnalyzer (la)

**LogAnalyzer (la)** — продукт для инженеров технической поддержки и аналитиков:
загрузка, разбор и визуализация лог-файлов заказчиков.

- **Загрузка** одного или нескольких лог-файлов либо zip-архива одновременно
  (drag&drop, мульти-выбор).
- **Автоопределение формата** и разбор в структурированные записи (Oracle alert,
  WebLogic, WLS stdout, Java/log4j, access, ODL + fallback `text`).
- **Просмотр**: таблица загрузок, drill-in, селектор файлов, мульти-поиск по
  полям, таймлайн, подкраска строк, пофайловые таблицы, график событий
  (месяц/день/час/минута).
- **Постобработка**: каждый файл получает сводку (число записей, размер, кодировка,
  first/last ts, длительность, level_counts, сессии старт-стоп).
- **Плагины** парсеров и постобработчиков (Go `.so`) — добавление/модификация без
  пересборки бинарника и без повторного развёртывания (только модуль + рестарт).
- **Поставка** — ZIP-архив, без инсталлятора; в runtime один процесс backend
  раздаёт API и статический frontend; **Node.js в runtime не требуется** (Angular
  компилируется build-time, в поставке — собранный JavaScript).

## Состав дистрибутива

```
la-<version>/
├── devagent                 # бинарник backend (Go)
├── la.conf.template         # шаблон конфигурации (для справки)
├── frontend/browser/...      # собранный Angular (статика)
├── parsers/*.so              # плагины-парсеры
├── postprocessors/*.so      # плагины-постобработчики
├── sh/
│   ├── start-la.sh           # старт
│   ├── stop-la.sh            # останов
│   └── status-la.sh          # статус
└── docs/
    ├── README.md             # этот файл
    ├── INSTALL.md            # установка/развёртывание
    └── USAGE.md             # использование
```

## Быстрый старт

1. Распакуйте ZIP в любой каталог.
2. `sh/start-la.sh` (при первом старте создастся `la.conf` и SQLite-БД `la.db`).
3. Откройте в браузере `http://localhost:8888`.

Подробности — в [docs/INSTALL.md](docs/INSTALL.md) и [docs/USAGE.md](docs/USAGE.md).

## Требования к окружению

- ОС: **Linux** (Go-плагины `.so` поддерживаются только на Linux).
- Внешних зависимостей нет: Node.js, БД-сервер, инсталлятор — не требуются.
  SQLite-драйвер (pure-Go) и Angular-сборка входят в поставку.

## Архитектура (кратко)

Один процесс backend: REST API (`/api/*`) + статический frontend из одного
источника. Источник данных — SQLite (MVP; Postgres — позже, отдельной ЮС).
Парсеры и постобработчики — подключаемые модули; встроенный `text`-парсер и
базовый постобработчик всегда есть в хосте (fallback).
