# US-0003 — Просмотр и анализ логов — frontend (Angular)

## Как инженер тех.поддержки/аналитик
## я хочу в веб-интерфейсе просматривать загруженные логи: таблицу загрузок, drill-in в просмотр с селектором файлов, мульти-фильтром поиска, таймлайном, подкраской строк и графиками событий
## чтобы быстро находить аномалии/всплески и вести анализ без возни с сырыми логами.

## Контекст

Frontend-поток продукта LogAnalyzer. Angular-приложение компилируется build-time (Angular CLI → `dist/`), в runtime раздаётся backend (Go `embed.FS`/`http.FileServer`; Python `StaticFiles`); **Node в runtime не нужен**.
Состояние просмотра (фильтры, подкраска, границы таймлайна, выбор файлов) живёт на сервере в таблицах `t_view_filters`/`t_view_highlights` (US-0004) — survives restarts. Бэкенд-эндпоинты — в `architect/specs/viewer.spec.md` (часть переиспользует US-0002, часть новая — US-0004).

## Functional requirements (ФР)

1. **Таблица загрузок** (сортируемая, фильтруемая) с удалением файла и всех результатов его обработки. Поля: размер, дата загрузки, сводка из парсинга (если файл — один лог) / число файлов в архиве (если архив). Под таблицей — общая информация: размер хранилища, число загрузок и т.п.
2. **Drill-in в просмотр:** выбрав загрузку, пользователь проваливается в просмотр; в шапке — детали архива или детали файла.
3. **Селектор файлов** со списком файлов и чекбоксами; по умолчанию выбраны все файлы.
4. **Поле поиска:** фильтрует по вхождению текста в файле по всем полям (возможно одно дополнительное исходное `raw`-поле); поиск во всех файлах файлового селектора; фильтров может быть более одного; каждый фильтр удаляется независимо от других.
5. **Таймлайн-селектор** (если применимо): минимальная/максимальная дата, с возможностью двигать границы селектора.
6. **Подкраска строк:** поле ввода текста (с опциональным выбором кластерных лексем лога) и виджет выбора цвета — все подходящие строки подкрашиваются.
7. **Персистентность:** все результаты фильтрации и подкраски сохраняются между рестартами процесса (в таблицах). Очистка форматирования и фильтров — по отдельной кнопке.
8. **Пофайловые таблицы:** каждый файл показывается в отдельной таблице по 10 строк (или динамически изменяемо), таблицы идут одна за другой; над таблицей — название файла и краткая сводка мелким читаемым шрифтом. Таблицу можно закрыть — тогда чекбокс снимается в селекторе; если чекбокс отмечен — таблица показывается вновь.
9. **Новое окно (если возможно):** каждый файл можно открыть в новом окне, но с управлением от основной страницы; если таблица открыта в отдельном окне — на основной странице она более не показывается.
10. **График событий:** дополнительно по каждой таблице может быть показан график числа событий с переключаемой группировкой месяц/день/час/минута — чтобы видеть всплески событий.

## Acceptance criteria

- [ ] AC-1: Таблица загрузок — сортируемая/фильтруемая; колонки размер, дата загрузки, сводка (один лог) / число файлов (архив); удаление строки с каскадным удалением файла и результатов; под таблицей — агрегаты (размер хранилища, число загрузок, файлов, записей).
- [ ] AC-2: Выбор строки загрузки → переход в просмотр; шапка показывает детали архива (zip: число файлов, общий размер, first/last ts) или детали одиночного файла.
- [ ] AC-3: Селектор файлов со списком файлов и чекбоксами; по умолчанию выбраны все.
- [ ] AC-4: Мульти-фильтр поиска по вхождению текста по всем полям (+ опц. исходное `raw`-поле); поиск по всем файлам селектора; несколько фильтров, каждый удаляется независимо.
- [ ] AC-5: Таймлайн-селектор по min/max дате с перемещаемыми границами (если применимо к выборке).
- [ ] AC-6: Подкраска строк: ввод текста + опц. выбор кластерных лексем + виджет цвета; все подходящие строки подкрашиваются.
- [ ] AC-7: Фильтры и подкраска сохраняются между рестартами (через backend-таблицы); отдельная кнопка очищает все фильтры и форматирование.
- [ ] AC-8: Каждый файл — отдельная пагинируемая таблица (10 строк / динамически), с заголовком (имя файла + краткая сводка мелким шрифтом); закрытие таблицы снимает чекбокс в селекторе, отметка — возвращает таблицу.
- [ ] AC-9: При возможности файл открывается в новом окне с управлением от основной страницы; открытое в окне не показывается на основной (graceful fallback, если окна невозможны).
- [ ] AC-10: По каждой таблице — график числа событий с переключаемой группировкой месяц/день/час/минута (всплески).
- [ ] AC-11: Сборка Angular — build-time (`ng build` → `dist/`); в runtime Node не требуется; `ng build`/`ng test` зелёные.

## Non-functional requirements

- Angular — только build-time (Node при сборке → `dist/`); в поставку входит собранный JavaScript, **Node не поставляется и не устанавливается** на ПК пользователя (см. NFR поставки US-0002/CLAUDE.md).
- Состояние просмотра — на сервере (таблицы `t_view_filters`/`t_view_highlights`), survives restarts; не единственный источник — localStorage.
- Пофайловые таблицы — пагинация (`limit`/`offset`), не грузить все записи в браузер.
- Мульти-файловый поиск/гистограмма — серверная агрегация; лимиты по числу файлов/записей.
- Python-релиз — полный функционал (те же эндпоинты/таблицы) по запросу.

## Зависимости

- US-0002 (бэкенд ингестии: таблицы/эндпоинты/сводки постобработки) — переиспользуются и расширяются.
- US-0004 (бэкенд-поддержка просмотрщика: мульти-файловый поиск, границы таймлайна, кластерные лексемы, персистентность фильтров/подкраски, гистограмма событий, агрегаты) — новые эндпоинты + миграция 0003.

## Открытые вопросы (уровень спецификации)

- Состав кластерных лексем (алгоритм) — закрыт на уровне MVP в US-0004 (топ-N токенов); уточняется по реализации.
- Поведение «новое окно» при невозможности (мобильные/блокировщики) — graceful fallback: таблица остаётся на основной странице (в спецификации).
- Авторизация/многопользовательность — пока single-tenant; view-state per upload (не per user).
- Полнотекстовый поиск (FTS5) — пока `LIKE`; FTS — отдельная ЮС.
